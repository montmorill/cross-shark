# MCN出海内容管理平台 - 需求文档 (PRD)

**版本**: v1.0
**日期**: 2026-02-10
**状态**: 初稿

---

## 一、项目概述

一站式MCN出海内容管理平台，支持从国内平台（B站/抖音/小红书）自动采集内容或手动上传，经过平台内管理和编排后，自动发布至海外社交平台（首期支持Twitter/X），同时提供账号数据监控和IP隔离能力，满足MCN机构批量运营海外账号的需求。

---

```
┌─────────────────────────────────────────────────────────┐
│                     MCN管理后台（Web）                      │

├──────────┴──────────┴───────────┴───────────┴───────────┤

├──────────┬──────────┬───────────┬───────────────────────┤

├──────────┴──────────┴───────────┴───────────────────────┤
│                    IP代理池 / 隔离层                        │
├─────────────────────────────────────────────────────────┤
│         B站 / 抖音 / 小红书    ←→    Twitter/X（首期）     │
│                                ←→    YouTube / TikTok（后续）│
└─────────────────────────────────────────────────────────┘
```

## 三、功能模块详细需求

### 模块一：内容采集

#### 1.1 自动爬取（无需登录）

**输入**: 目标平台 + 创作者ID
**支持平台**: B站（UID）、抖音（抖音号/分享链接）、小红书（小红书号）

| 功能点           | 说明                                     | 优先级 |
| ---------------- | ---------------------------------------- | ------ |
| 输入创作者ID     | 支持UID/主页链接/分享链接等多种格式      | P0     |
| 自动解析作品列表 | 获取该创作者的全部公开作品（视频+图文）  | P0     |
| 增量爬取         | 已爬取的不重复抓取，仅抓取新内容         | P0     |
| 定时爬取         | 设定频率自动检查更新（如每小时/每天）    | P1     |
| 内容下载         | 下载视频原文件、封面图、标题、描述、标签 | P0     |
| 去水印处理       | 自动去除平台水印（抖音/小红书）          | P1     |
| 爬取状态监控     | 显示爬取进度、成功/失败状态              | P0     |
| 反爬应对         | IP轮换、请求频率控制、UA伪装等           | P0     |

**采集的数据字段**:

- 视频/图片文件（原始质量）
- 标题
- 描述/正文
- 标签/话题
- 发布时间
- 点赞/评论/转发/收藏数（采集时的快照）
- 封面图
- 创作者信息（昵称、头像、粉丝数）

#### 1.2 手动上传

| 功能点     | 说明                                          | 优先级 |
| ---------- | --------------------------------------------- | ------ |
| 批量上传   | 支持拖拽或选择多个文件同时上传                | P0     |
| 支持格式   | 视频: mp4/mov/avi/mkv; 图片: jpg/png/gif/webp | P0     |
| 元信息填写 | 上传时可填写标题、描述、标签                  | P0     |
| 大文件支持 | 支持分片上传，断点续传（视频可能很大）        | P1     |
| 文件夹上传 | 支持整个文件夹批量导入                        | P2     |

**必须覆盖Twitter原生全部发布功能**:

| 功能点        | 说明                           | 优先级 |
| ------------- | ------------------------------ | ------ |
| 发推文        | 纯文本推文（280字符限制）      | P0     |
| 图片推文      | 支持1-4张图片                  | P0     |
| 视频推文      | 支持视频上传发布               | P0     |
| GIF推文       | 支持GIF发布                    | P1     |
| 投票推文      | 创建带投票的推文               | P2     |
| 长推文/Thread | 支持发布推文串（多条连续推文） | P1     |
| 引用推文      | 引用转发功能                   | P2     |
| 回复推文      | 对指定推文进行回复             | P2     |
| 定时发布      | 指定精确时间发布               | P0     |
| 标签/Hashtag  | 自动添加或手动编辑hashtag      | P0     |
| @提及         | 支持@其他用户                  | P1     |
| 敏感内容标记  | 按需标记敏感内容               | P2     |

#### 3.2 发布策略

| 策略类型     | 说明                                   | 优先级 |
| ------------ | -------------------------------------- | ------ |
| 定时发布     | 指定具体日期和时间                     | P0     |
| 定量发布     | 设定每天/每周发布数量上限              | P0     |
| 间隔发布     | 每隔N小时/N分钟发一条                  | P0     |
| 最佳时间发布 | 根据时区和粉丝活跃度自动选择发布时间   | P2     |
| 随机化       | 发布时间加入随机偏移，避免机器行为检测 | P1     |
| 队列管理     | 可排序的发布队列，支持插队/暂停/恢复   | P0     |
| 发布确认     | 发布前预览，可选人工审核               | P1     |
| 失败重试     | 发布失败自动重试，可配置重试次数和间隔 | P0     |

#### 3.3 发布队列

```
素材库 → [加入队列] → 发布队列（排序/编辑） → [策略执行] → 发布到Twitter
                         ↑                              ↓
                    手动调整顺序                    发布结果记录
                    暂停/恢复                      成功/失败状态
                    移除/插队
```

| 功能点     | 说明                                      | 优先级 |
| ---------- | ----------------------------------------- | ------ |
| 添加到队列 | 从素材库选择单个/批量添加到指定账号的队列 | P0     |
| 队列排序   | 拖拽排序，支持置顶/插队                   | P0     |
| 队列预览   | 预览即将发布的内容和计划时间              | P0     |
| 暂停/恢复  | 一键暂停某账号的发布队列                  | P0     |
| 批量操作   | 批量移除、批量调整发布时间                | P1     |
| 跨账号分发 | 同一内容分发到多个账号（可设不同时间）    | P1     |

### 模块四：账号管理

#### 4.1 账号体系

| 功能点   | 说明                                         | 优先级 |
| -------- | -------------------------------------------- | ------ |
| 添加账号 | 支持添加Twitter账号（OAuth授权或Cookie导入） | P0     |
| 账号分组 | 按项目/客户/类型分组管理                     | P1     |
| 账号状态 | 正常/受限/封禁/未验证 状态标识               | P0     |
| 账号信息 | 显示头像、昵称、粉丝数、发帖数等基础信息     | P0     |
| 批量管理 | 批量添加、批量编辑分组                       | P1     |

#### 4.2 IP隔离（核心安全功能）

| 功能点     | 说明                                         | 优先级 |
| ---------- | -------------------------------------------- | ------ |
| 独立IP绑定 | 每个Twitter账号绑定独立的代理IP              | P0     |
| IP池管理   | 管理代理IP池，支持添加/删除/测试连通性       | P0     |
| 代理类型   | 支持HTTP/HTTPS/SOCKS5代理                    | P0     |
| 自动分配   | 新账号自动分配未使用的IP                     | P1     |
| IP健康检测 | 定期检测代理IP可用性和速度                   | P0     |
| 指纹隔离   | 不同账号使用不同的浏览器指纹（UA、分辨率等） | P1     |
| 隔离检测   | 检测同IP下是否有多个账号（防关联）           | P0     |

**IP隔离架构**:

```
账号A ──→ 代理IP-1 (美国) ──→ Twitter API
账号B ──→ 代理IP-2 (日本) ──→ Twitter API
账号C ──→ 代理IP-3 (英国) ──→ Twitter API
监控服务 ──→ 独立代理池 ──→ Twitter（数据抓取）
```

**关键原则**: 发布用代理 和 监控用代理 必须完全隔离，不共用IP。

## 四、技术要点与风险

### 4.1 内容采集风险

| 风险                    | 应对方案                                    |
| ----------------------- | ------------------------------------------- |
| B站/抖音/小红书反爬升级 | 多种爬取方案（API+网页+App抓包），保持更新  |
| 平台接口变动            | 模块化设计，各平台爬虫独立，便于快速修复    |
| 内容版权问题            | 由MCN用户自行负责版权合规，平台提供免责声明 |

### 4.2 Twitter发布风险

| 风险          | 应对方案                                  |
| ------------- | ----------------------------------------- |
| 账号被封/限流 | IP隔离 + 行为模拟（随机间隔、人类化操作） |
| API限制       | 合理使用API额度，超限时降级到网页自动化   |
| 内容重复检测  | 支持内容微调（标题改写、描述差异化）      |
| OAuth授权过期 | 自动刷新Token，异常时通知用户             |

### 4.3 数据监控风险

| 风险            | 应对方案                               |
| --------------- | -------------------------------------- |
| Twitter反爬封IP | 独立IP池、频率控制、多IP轮换           |
| 数据不完整      | 多来源交叉验证，API+爬虫双链路         |
| 数据延迟        | 最低保证每日一次，核心指标支持更高频率 |

## 五、开发阶段规划

- [ ] 内容采集：B站爬取 + 手动上传

### Phase 3：高级功能

- [ ] Thread发布、投票、引用等Twitter高级功能

- [ ] 数据对比 + 导出 + 告警

## 六、待确认事项

1. **技术栈选择**: 前端框架（React/Vue）、后端语言（Python/Node/Go）、数据库
2. **部署方式**: 本地部署 / 云服务器 / SaaS化
